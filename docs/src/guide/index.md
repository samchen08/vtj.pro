# 关于 VTJ.PRO

## 我们为什么做 VTJ.PRO？

低代码火了很多年，但大多数产品的逻辑是：  
**用“效率”换“控制权”。**

- 拖出来的页面，是一坨不可读的代码
- 项目一旦做大，就必须依赖平台
- 想二次开发？对不起，不支持
- 想私有化？价格贵到离谱

作为开发者，我们很清楚这意味着什么：  
**短期爽，长期痛。**

VTJ.PRO 就是在这个背景下诞生的。

## VTJ.PRO 是什么？

> **VTJ.PRO 是给专业开发者的 AI 低代码引擎。**

它不是“业务人员的零代码工具”，  
也不是“只能在线使用的 SaaS 平台”，  
而是一个**可私有化、可深度定制、可完全掌控源码的工程化引擎**。

我们用一句话概括它：

> **设计即代码，代码即设计。**

## 我们的核心原则

### 1️⃣ 绝不锁代码

VTJ.PRO 生成的是**标准 Vue 3 + TypeScript 项目**：

- 无私有运行时
- 无平台专属标记
- 无“离开平台就废”的黑盒逻辑

你可以：

- 在设计器里拖
- 在 IDE 里改
- 随时脱离 VTJ.PRO 继续维护

**这是我们对开发者最基本的尊重。**

### 2️⃣ 双向代码转换

这是我们和绝大多数低代码平台最大的区别。

- 拖拽生成的页面 → 导出为 Vue 单文件组件
- 已有的 Vue 项目 → 反向解析为 DSL，回到设计器继续编辑

这不是“代码预览”，而是**真正的双向编译闭环**。

### 3️⃣ AI 是副驾驶，不是玩具

VTJ.PRO 里的 AI 不是为了“聊天好看”，而是为了**干活**：

- 自然语言生成页面 / 组件
- 设计稿 → Vue 组件
- 报错信息 → 自动修复代码
- 选中组件 → AI 帮你改样式、补逻辑

我们希望你用 AI，是为了**少写重复代码，而不是多学一个新系统**。

### 4️⃣ 工程化是第一优先级

VTJ.PRO 从第一天起就按企业级项目标准设计：

- Vue 3 / Vite / TypeScript
- Pinia / vue-i18n / ESLint
- 多端输出：Web / H5 / UniApp
- Docker 私有化部署

**我们不教你“新框架”，我们只让你用你已经熟悉的东西，更快一点。**

## 谁在用 VTJ.PRO？

VTJ.PRO 适合：

- 需要快速交付中后台系统的团队
- 外包公司 / 内部工具团队
- 想把老旧 Vue 项目现代化改造的技术负责人
- 对代码质量有执念的前端工程师

我们不强求“人人都用”，  
我们只服务那些**既想要效率，又不想丢掉控制权的人**。
